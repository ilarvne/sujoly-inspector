<objective>
Implement embedding generation service and wire it into the data pipeline so structures, inspections, and documents get vector embeddings for semantic search.

Purpose: The hybrid search endpoint needs vector embeddings to work. This plan creates an embedding service that generates embeddings for text content (structure names/descriptions, inspection findings, document text) and stores them in the embeddings table. A Celery task handles batch embedding generation.

Output: EmbeddingService + Celery task for embedding generation. Structures/inspections get auto-embedded on creation.
