## Plan 03 Slice: OGC API for External GIS Clients

**As a** GIS analyst, **I want to** connect QGIS to the OGC API Features/Tiles endpoint and load hydraulic structures with CQL2 filtering, **so that** I can view and analyze structures in my GIS workflow.

<objective>
Add TiPG as a standalone Docker container (D-04) that auto-discovers the structures table from PostgreSQL and serves OGC API Features (Part 1 Core, Part 3 Filtering/CQL2) and OGC API Tiles (Part 1 Core) for external GIS clients like QGIS and ArcGIS (INT-01). TiPG requires no custom code — pure configuration via environment variables (D-06). Integration tests verify the OGC API endpoints respond correctly, including CQL2 filtering and NULL geometry handling (D-02).

Purpose: This plan delivers INT-01 — OGC API Features/Tiles access for external GIS clients. It depends on Plan 01 for the schema (geometry column exists for TiPG auto-discovery) and ingested data.
Output: TiPG service in docker-compose.yml, .env.example TiPG vars, test_tipg.py integration tests
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External GIS client → TiPG OGC API | Untrusted CQL2 filter expressions and query parameters cross here |
| TiPG → PostgreSQL | Direct database connection (internal, TiPG uses parameterized queries) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02 | Tampering | TiPG CQL2 filter parameter | accept | TiPG uses a proper CQL2 parser (not string interpolation) to parse filter expressions. CQL2 is a structured query language with defined grammar. The parser prevents SQL injection by constructing parameterized queries internally. [CITED: RESEARCH.md Security Domain, TiPG factory.py source] |
| T-02-05 | DoS | TiPG items endpoint | mitigate | TIPG_MAX_FEATURES_PER_QUERY=10000 limits maximum features returned per query. TiPG also supports limit/offset pagination. This prevents unbounded result sets from consuming database resources. |
| T-02-06 | Information Disclosure | TiPG error responses | accept | TiPG returns OGC-compliant error responses. No SQL queries or internal paths are exposed in error messages. Database connection details are in environment variables, not exposed via API. |
| T-02-SC | Tampering | TiPG Docker image | mitigate | Image ghcr.io/developmentseed/tipg:latest is the official Development Seed image from GitHub Container Registry. Package Legitimacy Audit in RESEARCH.md confirms Approved. No [SLOP] or [SUS] verdict. |
</threat_model>
