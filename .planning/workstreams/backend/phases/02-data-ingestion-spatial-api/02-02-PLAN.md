## Plan 02 Slice: REST API + Multilingual Search

**As a** frontend developer, **I want** REST API endpoints to list, search, filter, and manage structures with multilingual full-text search, **so that** the frontend can display and interact with the structure catalog.

<objective>
Create the REST API layer for structures following the existing provenance route/service/model pattern. Pydantic schemas define request/response models. The service layer implements async CRUD operations with SQLAlchemy 2.0, multilingual FTS using generated tsvector columns (D-10) combined with pg_trgm fuzzy matching (D-11, D-12), attribute filtering (D-08), bbox spatial filtering, and pagination with total count (D-16). Routes expose standard CRUD + a dedicated search endpoint (D-13, D-14). GeoJSON response format option for map clients (D-16).

Purpose: This plan delivers DATA-08 (multilingual search + filter) and the CRUD/search portion of INT-03 (REST API for frontend). It depends on Plan 01 for the schema (tsvector columns, trigram indexes, filterable columns) and ingested data.
Output: schemas/structures.py, structure_service.py, routes/structures.py, test_structures.py
</objective>

Create apps/api/src/api/services/structure_service.py with async CRUD + search functions following the provenance_service.py pattern (structlog logger, async with async_session() as session, async with session.begin() for writes, select() with .where() for queries):

create_structure(data: StructureCreate, provenance_id: uuid.UUID) -> StructureModel — async with async_session() as session, async with session.begin(), create StructureModel with all fields from data + provenance_id, session.add, await session.flush, await session.refresh, return model.

get_structure(structure_id: uuid.UUID) -> StructureModel | None — select(StructureModel).where(StructureModel.id == structure_id), return scalar_one_or_none.

list_structures(filters: dict, q: str|None, lang: str, bbox: str|None, offset: int, limit: int, format: str) -> tuple[list, int] — build select(StructureModel) stmt. Apply attribute filters per D-16: if filters["type"], stmt = stmt.where(StructureModel.type == filters["type"]). Same for district, technical_condition, water_source. Apply bbox spatial filter per D-14: if bbox provided, parse to floats (reject non-numeric — T-02-03 mitigation), validate exactly 4 values, use func.ST_MakeEnvelope(minx, miny, maxx, maxy, 4326) and geofunc.ST_Intersects(StructureModel.geometry, envelope) with StructureModel.geometry.isnot(None) to skip NULL geometries. Compute total count via select(func.count()).select_from(stmt.subquery()). Apply offset/limit. Return (items, total). If q is provided, delegate to search_structures. The bbox filtering pattern is in RESEARCH.md lines 836-876.

search_structures(q: str, lang: str, filters: dict, bbox: str|None, limit: int) -> tuple[list, int] — implement combined FTS + trigram search per D-12. Select the appropriate tsvector column based on lang: "ru" → search_ts_ru with to_tsvector('russian', ...), "kk" → search_ts_kk with to_tsvector('simple', ...), "en" → search_ts_en with to_tsvector('english', ...). Use func.ts_rank_cd for FTS ranking and func.similarity for trigram fuzzy matching across name_ru, name_kk, name_en. Compute blended_score = fts_rank * 0.7 + greatest(similarity scores) * 0.3 per RESEARCH.md lines 307-321. WHERE clause: tsvector @@ plainto_tsquery OR name_ru % q OR name_kk % q OR name_en % q (using pg_trgm % operator with GIN index). Apply attribute filters and bbox same as list_structures. Order by blended_score DESC. Return (items_with_score, total). Use SQLAlchemy text() or func constructs — never string-interpolate user input (T-02-01 mitigation). The combined search SQL is in RESEARCH.md lines 307-321.

update_structure(structure_id: uuid.UUID, data: StructureUpdate) -> StructureModel | None — fetch existing, update non-None fields, return updated model. Per D-13, this creates new structure_facts with new provenance and expires old facts (set valid_to on old facts, create new facts with new provenance_id). For Phase 2, a simple field update on the StructureModel is sufficient — the provenance-per-fact update pattern can be refined later.

delete_structure(structure_id: uuid.UUID) -> bool — soft delete: set status="deleted" per D-13 ("soft delete via status field"). Return True if found, False if not.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → FastAPI structures endpoints | Untrusted query parameters (q, bbox, filters) and request bodies cross here |
| FastAPI → PostgreSQL | SQLAlchemy parameterized queries (internal, parameterized by ORM) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | search_structures query construction | mitigate | Use SQLAlchemy ORM select().where() with parameterized values for all user input. Never use string interpolation or text() with f-strings for the q parameter. Use func.plainto_tsquery() and func.similarity() via SQLAlchemy function constructs which are parameterized. |
| T-02-03 | Tampering | bbox parameter in list_structures | mitigate | Parse bbox string to floats with float() — reject non-numeric input with HTTPException 400. Validate exactly 4 values (minx, miny, maxx, maxy). Use func.ST_MakeEnvelope with parameterized float values, not string interpolation. Skip NULL geometries with StructureModel.geometry.isnot(None). |
| T-02-05 | DoS | GET /structures and GET /structures/search | mitigate | limit parameter constrained to max 1000 (Query(100, ge=1, le=1000) for list, Query(20, ge=1, le=100) for search). Prevents unbounded result sets. |
| T-02-06 | Information Disclosure | Error responses | accept | Global exception handler from Phase 1 returns generic 500 for unhandled errors. HTTPException returns structured error detail (safe — no SQL or internal paths exposed). Structured logging captures details server-side. |
</threat_model>
