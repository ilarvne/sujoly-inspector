## Phase Goal

**As a** hydraulic structures manager, **I want to** compute risk-informed inspection intervals from structure condition and contextual factors, **so that** I can prioritize inspections based on defensible, explainable risk scores.

> Note: ROADMAP goal line is not in user-story format. User story derived from Success Criteria #1 and #2.

<objective>
Implement the risk computation engine as a pure Python module (D-01) with a semi-quantitative formula (D-02/D-03), red-flag overrides (D-07), four repair statuses (D-08), and weak-evidence floor (D-09). This module is the analytical core of Phase 3 — it takes structure data as dict input and returns a RiskAssessment with full factor breakdown for explainability.

Purpose: The risk engine is the most complex and most testable component. Isolating it as a pure Python module (no DB, no async, no side effects) enables comprehensive unit testing and ensures the computation is defensible per the PROJECT.md principle "Semi-quantitative risk index over black-box ML."

Output: `apps/api/src/api/services/risk_engine.py` (pure Python module) + `apps/api/tests/test_risk_engine.py` (comprehensive unit tests covering RISK-01 through RISK-05)
</objective>
