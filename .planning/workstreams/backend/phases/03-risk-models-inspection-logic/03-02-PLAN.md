## Phase Goal

**As a** system administrator, **I want to** authenticate users with role-based access control and enforce the permissions matrix on all structure endpoints, **so that** inspectors, engineers, and admins have appropriate permissions for risk overrides, inspection creation, data export, and structure management.

<objective>
Implement JWT-based authentication with four-role RBAC (D-10, D-11, D-12), users table with admin seeding (D-10), FastAPI auth dependencies for role enforcement, retrofit RBAC on existing structure endpoints per D-12 permissions matrix (create/update require engineer+, delete requires admin), and all infrastructure prerequisites for Phase 3 (PyJWT/WeasyPrint/Jinja2 dependencies, JWT settings, celery-beat service, router auto-discovery).

Purpose: RBAC is the security foundation for all Phase 3 endpoints. Every subsequent plan (03–06) depends on `require_role()` and `get_current_user` from this plan. The D-12 permissions matrix must be enforced on existing structure endpoints (create/update/delete) — these were open in Phase 2 and need require_role dependencies retrofitted. Infrastructure changes (dependencies, settings, docker-compose) are bundled here to avoid file conflicts in later plans.

Output: Working auth endpoints (POST /auth/token, GET /auth/me), `require_role()` dependency factory used by all protected endpoints, users table with admin seed, RBAC enforcement on existing structure CRUD endpoints, all Phase 3 dependencies installed.
</objective>
