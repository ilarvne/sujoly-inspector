<objective>
Enforce RBAC on all read endpoints and harden authentication so role permissions are meaningfully enforced.

Purpose: Three BLOCKER/WARNING gaps prevent RBAC from being effective: (1) GET structure/risk endpoints are fully public, (2) username-only token issuance allows trivial impersonation, (3) docker-compose ships with hardcoded secrets. This plan closes all three by wiring require_role('viewer') to read paths, requiring api_key alongside username for tokens, hashing API keys at rest, and removing insecure defaults.

Output: Authenticated-only API with enforced RBAC on all endpoints, hardened token issuance, and secure-by-default deployment configuration.
</objective>

    **Gaps closed:** WARNING — "Authentication trivially bypassable" (#4) + "Docker secrets insecure fallbacks" (#5)

    **A. Require api_key alongside username for token issuance:**

    Change `schemas/auth.py` TokenRequest: both `username` and `api_key` become required fields (not Optional). Remove the `username`-only path.

    Change `routes/auth.py` token_endpoint:
    - Accept username + api_key
    - Look up user by username via `auth_service.get_user_by_username`
    - Verify the provided api_key against the stored hash via `auth_service.verify_api_key(user, provided_api_key)`
    - Only issue JWT if both username exists AND api_key matches

    **B. Hash API keys at rest:**

    Add `api_key_hash: Mapped[str | None]` column to UserModel (String 64, nullable, indexed). The hash is SHA-256 of the api_key value.

    Add to `services/auth_service.py`:
    - `hash_api_key(api_key: str) -> str`: returns `hashlib.sha256(api_key.encode()).hexdigest()`
    - `verify_api_key(user: UserModel, provided_key: str) -> bool`: hashes provided_key and compares with `hmac.compare_digest(user.api_key_hash, hash_api_key(provided_key))`
    - Update `get_user_by_api_key` to iterate candidates and use `verify_api_key` (or, more efficiently, hash the input and query by `api_key_hash`)

    Update `main.py` lifespan admin seeding: hash the `settings.initial_admin_api_key` before storing in `UserModel.api_key_hash`.

    **C. Create Alembic migration 0007_user_api_key_hash:**

    Follow pattern from 0003_users.py:
    - `op.add_column("users", sa.Column("api_key_hash", sa.String(64), nullable=True))`
    - `op.create_index("ix_users_api_key_hash", "users", ["api_key_hash"])`
    - Data migration: for each user with non-null `api_key`, compute `hashlib.sha256(api_key.encode()).hexdigest()` and set `api_key_hash`
    - Keep existing `api_key` column for backward compatibility (removal deferred)

    **D. Remove insecure Docker defaults:**

    In `docker-compose.yml`, change all three services (api, celery-worker, celery-beat):
    - `API_JWT_SECRET: ${API_JWT_SECRET}` — remove `:-dev-secret-change-me` fallback

    - If validation fails, raise `ValueError("API_JWT_SECRET must be set and at least 32 characters")`
    - In development mode (`self.environment == "development"`), allow override with a warning log instead of raising

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API (unauthenticated) | Untrusted requests to read endpoints now rejected with 401 |
| Client → POST /auth/token | Untrusted credentials; must verify both username AND api_key |
| API → Database | API key hashes stored; no plaintext secrets in DB |
| Docker env → API | Secrets must be explicitly provided; no insecure defaults |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07a | Spoofing | POST /auth/token | mitigate | Require both username AND api_key; reject username-only requests (closes trivial impersonation) |
| T-03-07b | Tampering | JWT token | mitigate | Validate JWT_SECRET >= 32 chars on startup; remove fallback defaults from docker-compose |
| T-03-07c | Information Disclosure | users table | mitigate | Store api_key_hash (SHA-256) instead of plaintext api_key; use hmac.compare_digest for constant-time comparison |
| T-03-07d | Spoofing | GET read endpoints | mitigate | Add require_role('viewer') to all read endpoints; unauthenticated access returns 401 |
| T-03-07e | Elevation of Privilege | RBAC bypass | mitigate | Close username-only auth path; require secret knowledge (api_key) for any token issuance |
</threat_model>
