## Phase Goal

**As an** inspector, **I want to** attach scanned passports, inspection reports, and photos to structure records, **so that** all documentary evidence is linked to the canonical structure record and accessible via secure presigned URLs.

<objective>
Implement document attachment CRUD endpoints with MinIO integration (D-17, D-18). Documents are stored in MinIO `sujoly-documents` bucket (existing from Phase 1). The API stores only metadata + object keys in PostgreSQL per the INT-04 architecture separation principle.

Purpose: DATA-06 requires document attachment endpoints via MinIO presigned URLs. This reuses the existing MinIOService presigned URL pattern from Phase 1. Client flow: get presigned upload URL → upload to MinIO → register metadata via API.

Output: DocumentModel, document_service with CRUD + MinIO, document API endpoints with RBAC enforcement.
</objective>
