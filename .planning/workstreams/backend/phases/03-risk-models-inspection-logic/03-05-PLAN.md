## Phase Goal

**As an** inspector, **I want to** record inspection findings with photos for each structure, **so that** inspection history is maintained with condition data that triggers risk recomputation.

<objective>
Implement inspection history CRUD with photo attachments via MinIO (D-14, D-15, D-16). Inspections table stores findings, condition, and red_flags. Photos are linked via inspection_photos table with MinIO object keys. Creating an inspection triggers risk recomputation via Celery (D-05 trigger #1).

Purpose: DATA-05 requires inspection history per structure with date, inspector, findings, photo URLs, and condition. This plan delivers the complete inspection lifecycle: create with photos → list with presigned URLs → detail view. The risk recomputation trigger connects inspections to the risk engine (Plan 01) via risk_service (Plan 03).

Output: InspectionModel + InspectionPhotoModel, inspection_service with CRUD + photo linking + recomputation trigger, inspection API endpoints with RBAC.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API inspection endpoints | JWT required — inspector+ for create, all roles for list/detail |
| API → MinIO | Presigned URL generation for photo downloads |
| API → Celery broker | Risk recomputation task dispatched via .delay() |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-14 | Information Disclosure | Photo presigned URLs in list/detail responses | mitigate | Short expiry (2 hours via MinIOService.presigned_download_url). URLs not logged. Generated on-demand, not stored in DB. |
| T-03-15 | Elevation of Privilege | POST /structures/{id}/inspections | mitigate | require_role("inspector") dependency — viewer cannot create inspections per D-12. Test: test_create_inspection_viewer_forbidden asserts 403. |
| T-03-16 | Tampering | Inspection creation — mass assignment | mitigate | Explicit InspectionCreate Pydantic schema with typed fields. photos is list[PhotoMetadata] with Literal enum for photo_type. No arbitrary field injection. |
| T-03-17 | Tampering | Inspection photo object keys | accept | Photo object keys come from client after MinIO upload. The API stores keys as-is (no path traversal risk — MinIO handles object key validation). Object keys are not used for file system paths. |
</threat_model>
