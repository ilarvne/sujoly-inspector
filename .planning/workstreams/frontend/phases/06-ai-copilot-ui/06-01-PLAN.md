## Phase Goal

**As a** water resource manager or inspector, **I want to** ask natural-language questions about hydraulic structures and get evidence-grounded answers with source citations, **so that** I can quickly understand structure conditions, risks, and inspection history without manually browsing the map.

---

<objective>
Create the full data layer for the AI Copilot: types for chat messages/sources/cards, a mock AI engine with trilingual intent detection that queries existing mock data, a Zustand chat store with streaming simulation and localStorage persistence, and all trilingual i18n keys. This is the foundation that Plan 06-02 builds the UI upon.
</objective>

<description>
Add Copilot types to lib/api/types.ts. Create lib/copilot/mock-ai-engine.ts with trilingual intent detection and response generation. Create lib/stores/chat-store.ts with Zustand persist and streaming simulation. Add copilot i18n keys to all 3 message files. Add unit tests.
</description>
<instructions>
1. Add to lib/api/types.ts:
   - `ChatIntent` union type: 'list_critical' | 'list_repair' | 'list_inspection' | 'show_risk' | 'summarize_inspections' | 'explain_condition' | 'list_by_district' | 'list_by_basin' | 'general'
   - `CopilotSourceType`: 'inspection' | 'registry' | 'osm' | 'document' | 'risk_assessment'
   - `CopilotSource`: { id, type: CopilotSourceType, label, reference, structureId? }
   - `CopilotCardType`: 'structure' | 'risk' | 'inspection' | 'report'
   - `CopilotCard`: discriminated union with type field and data payload
   - `CopilotMessage`: { id, role: 'user'|'assistant', content, timestamp, sources?, cards?, isStreaming? }

2. Create lib/copilot/mock-ai-engine.ts:
   - Import mockStructures, mockStructureById, mockRiskScore, mockInspections from lib/api/mock-data
   - Import StructureDetail, RiskScore, InspectionRecord types
   - `detectIntent(query: string): ChatIntent` — keyword matching against RU/KK/EN dictionaries
   - `mockAIEngine(query: string, locale: 'ru'|'kk'|'en'): { text: string; sources: CopilotSource[]; cards: CopilotCard[] }` — generates full response
   - `getSuggestedPrompts(locale): string[]` — returns 4-5 trilingual suggested questions
   - Intent handlers for each ChatIntent type that query mock data and format responses in the locale language

3. Create lib/stores/chat-store.ts:
   - Zustand store with persist middleware (key: 'sujoly-chat')
   - State: messages: CopilotMessage[], isStreaming: boolean
   - Actions: sendMessage(content: string, locale), clearChat()
   - sendMessage: adds user message, creates assistant placeholder with isStreaming=true, calls mockAIEngine, simulates streaming with setTimeout chunks, attaches sources+cards, sets isStreaming=false
   - Use refs/cleanup to handle setTimeout properly

4. Add i18n keys to messages/{en,ru,kk}.json under 'copilot' namespace (expand existing):
   - chatPlaceholder, send, sendAria, suggestedPrompts (array of 4-5), clearChat, streaming, sourceLabel, sourcesTitle
   - Card titles: structureCard, riskCard, inspectionCard, reportCard
   - Source type labels: sourceInspection, sourceRegistry, sourceOsm, sourceDocument, sourceRisk
   - Empty state: noMessages, welcomeMessage
   - Error: errorMessage

5. Create tests/phase6-mocks.test.ts:
   - Test detectIntent with RU/KK/EN keywords for each intent
   - Test mockAIEngine returns text, sources, cards for each intent
   - Test getSuggestedPrompts returns correct prompts per locale
   - Test chat store sendMessage adds messages and sets isStreaming
   - Test chat store clearChat empties messages
