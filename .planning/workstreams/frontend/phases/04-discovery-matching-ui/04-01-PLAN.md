# Plan 04-01: Foundation

## Overview
Create the data layer (types, mock data, API hooks, Zustand store), i18n keys, and reusable confidence/source/evidence components for the Discovery & Matching UI.

## Tasks

<task id="04-01" title="Types, mock data, API hooks, store, i18n, components">
### Steps
1. Add discovery types to `lib/api/types.ts`:
   - `DiscoverySource` = 'osm' | 'satellite' | 'kazvodhoz' | 'manual'
   - `ConfidenceLevel` = 'high' | 'medium' | 'low'
   - `MatchEvidence` — { type, label, value, agreement: boolean }
   - `ReviewStatus` = 'pending' | 'accepted' | 'linked' | 'rejected'
   - `DiscoveryCandidate` — id, source, sourceName, coordinates, name, type, confidence, evidence[], detectedAt, properties
   - `MatchResult` — candidateId, existingStructureId (nullable), matchScore, evidence[], reviewStatus
   - `DiscoveryReviewAction` — candidateId, action: 'accept'|'link'|'reject', reviewerName, timestamp, reason

2. Add mock data to `lib/api/mock-data.ts`:
   - `mockDiscoveryCandidates()` — generate ~20 candidates from OSM/satellite sources
   - `mockMatchResults()` — generate match results linking candidates to existing structures
   - `mockReviewActions()` — initial empty, track review history

3. Add API hooks to `lib/api/client.ts`:
   - `useDiscoveryCandidates()` — useQuery
   - `useMatchResults()` — useQuery
   - `useReviewActions()` — useQuery
   - `useSubmitReview()` — useMutation (mock, updates local state)

4. Create `lib/stores/discovery-store.ts`:
   - selectedCandidateId, reviewFilter (all/pending/accepted/linked/rejected), searchQuery
   - setSelectedCandidate, setReviewFilter, setSearchQuery

5. Add i18n keys to messages/{en,ru,kk}.json under `discovery` namespace:
   - title, subtitle, candidateCount, confidence (high/medium/low)
   - sources (osm/satellite/kazvodhoz/manual)
   - evidence types (name_similarity, distance, type_agreement, source_overlap)
   - review (accept, link, reject, accepted, linked, rejected, pending)
   - comparison (existingRecord, candidateRecord, noMatch, matchScore)
   - table headers (source, name, type, confidence, status, actions)
   - filter labels (all, pending, reviewed)
   - messages (reviewSuccess, confirmAccept, confirmLink, confirmReject)

6. Create reusable components:
   - `components/discovery/confidence-badge.tsx` — Badge with color (green=high, yellow=medium, gray=low)
   - `components/discovery/source-chip.tsx` — Badge with icon + source name
   - `components/discovery/evidence-chip.tsx` — Badge with check/x icon + evidence label

### Verification
- `npm run build` passes in apps/web/
</task>
