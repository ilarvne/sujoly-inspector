# Plan 04-02: Discovery & Matching UI

## Overview
Build the full Discovery & Matching UI on the /hydrofinder route: candidate list table, side-by-side comparison view with evidence chips, and review workflow (accept/link/reject) with one-click actions.

## Tasks

<task id="04-02" title="Candidate list, comparison view, review actions, page wiring">
### Steps
1. Create `components/discovery/candidate-list.tsx`:
   - Table with columns: source, name, type, confidence, match score, status, actions
   - Filter by review status (all/pending/accepted/linked/rejected) via Select
   - Search by name via Input
   - Click row → selects candidate (sets in discovery store)
   - Confidence badges and source chips on each row
   - Uses useDiscoveryCandidates + useMatchResults

2. Create `components/discovery/comparison-view.tsx`:
   - Side-by-side: existing record (left) vs candidate (right)
   - Fields: name, type, coordinates, district, basin
   - Evidence chips between/below (name similarity %, distance km, type agreement)
   - Match score progress bar
   - "No existing match" state when matchScore is low or no existingStructureId
   - Uses useStructureDetail for existing record, useDiscoveryCandidates for candidate

3. Create `components/discovery/review-actions.tsx`:
   - Three buttons: Accept (add to registry), Link (merge with existing), Reject (false positive)
   - Accept/Reject always available; Link only when existingStructureId exists
   - Confirmation dialog with optional reason field
   - On submit: calls mockSubmitReview, shows success message
   - Permission guard: only admin/engineer can review
   - Status badge showing current review status

4. Create `components/discovery/discovery-view.tsx`:
   - Orchestrates: filter bar at top, two-column layout (candidate list left, comparison+review right)
   - Responsive: stacks on mobile
   - Stats summary: total candidates, pending, accepted, rejected counts

5. Update `app/[locale]/hydrofinder/page.tsx`:
   - Replace placeholder with full DiscoveryView component
   - Keep async Server Component pattern with getTranslations

### Verification
- `npm run build` passes in apps/web/
- Page renders at /ru/hydrofinder, /kk/hydrofinder, /en/hydrofinder
</task>
