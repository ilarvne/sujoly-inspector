# AGENT

Python 3.12 RAG agent: FastAPI + LangGraph + ChromaDB + Ollama/Mistral.

## STRUCTURE

```
src/agent/
├── config/          # Pydantic settings (env prefix: AGENT_)
├── core/            # Agent class — async context manager, PostgreSQL checkpointing
├── graphs/          # LangGraph StateGraph workflow (agent → tools → summarize)
├── infrastructure/  # SQLAlchemy async DB layer (documents table)
├── loaders/         # PDF/text ingestion with semantic chunking (800/200 split)
├── memory/          # ChromaDB vector store (singleton) + mem0 long-term memory
├── prompts/         # System + retrieval prompt templates (multilingual: RU/KZ)
├── retrieval/       # HybridRetriever: BM25 (SQLite FTS5) + dense + RRF + reranker
├── routes/          # FastAPI routers (agent, admin, health)
├── services/        # ConnectRPC AdminService implementation
├── tools/           # LangGraph @tool definitions + registry
├── utils/           # logging (structlog), observability (OTel), rate limiting (SlowAPI)
├── gen/             # Generated protobuf (DO NOT EDIT)
├── server.py        # FastAPI app — lifespan, middleware, route registration
├── server_auth.py   # Kratos session auth dependency
└── ingest.py        # CLI document ingestion entry point
tests/               # pytest (asyncio_mode = "auto")
```

## WHERE TO LOOK

| Task | Location | Notes |
|------|----------|-------|
| Add endpoint | `routes/` + register in `server.py` | FastAPI router with `@router` |
| Add tool | `tools/` + register in `tools/__init__.py` | `@tool` decorator, docstring = LLM description |
| Add graph node | `graphs/workflow.py` | StateGraph with AgentState |
| Modify retrieval | `retrieval/hybrid.py` | BM25 + dense → RRF fusion |
| Change prompts | `prompts/` | ChatPromptTemplate, multilingual keywords |
| Configuration | `config/settings.py` | Pydantic `AGENT_` prefixed settings |
| Database schema | `infrastructure/database.py` | SQLAlchemy async, DocumentModel |
| Document ingestion | `loaders/pdf.py`, `loaders/text.py` | Semantic chunking, dual-index (vector + BM25) |
| Memory | `memory/store.py` (vector), `memory/mem0_client.py` (LTM) | Thread-safe singletons |
| Auth dependency | `server_auth.py` | `get_current_user` — Kratos session validation |

## CONVENTIONS

- **Package manager**: `uv` (not pip), PyTorch CPU index for torch deps
- **Logging**: `structlog` JSON in prod — `from agent.utils import get_logger`
- **Telemetry**: OpenTelemetry FastAPIInstrumentor
- **Async**: pytest-asyncio `asyncio_mode = "auto"`
- **Barrel exports**: `__init__.py` with `__all__` for every module
- **Auth**: Kratos session validation via `server_auth.get_current_user` dependency
- **Rate limiting**: SlowAPI with custom key function (API key > user ID > IP)

## PATTERNS

### Settings Access
```python
from agent.config.settings import settings
settings.ollama_base_url  # Pydantic-validated, AGENT_ prefix
```

### Structured Logging
```python
from agent.utils import get_logger
logger = get_logger(__name__)
logger.info("event", key="value")  # structlog context
```

### Tool Definition
```python
@tool
def my_tool(query: str) -> str:
    """Tool docstring becomes LLM description."""
    return result
```

### Retrieval Pipeline
```
Query → BM25 (SQLite FTS5) ──┐
     → Dense (ChromaDB)  ────┼→ RRF Fusion → Cross-encoder Reranker → Results
```

### Graph Workflow
```
START → fetch_memory → agent → conditional(tools/summarize/END) → tools → agent → ...
```
Summarization triggers at >10 messages. Memory context injected from Mem0.

## ANTI-PATTERNS

- **NO** `print()` — use structlog logger
- **NO** editing `gen/` — regenerated by `bun run codegen`
- **NO** broad exception catches without logging
- **NO** direct ChromaDB access — use HybridRetriever
- **NO** hardcoded URLs — use settings/env vars

## NOTES

- Dual checkpointing: PostgreSQL primary, SQLite fallback
- Multilingual prompts: query expansion includes Russian/Kazakh translations
- Conversation summarization triggers at >10 messages
- `apps/agent/apps/` and `apps/agent/internal/` are buf artifacts — ignore
- Middleware chain: CORS → rate limit → request ID/timing headers → exception handler
- Reranker model: BAAI/bge-reranker-v2-m3 (cross-encoder)
