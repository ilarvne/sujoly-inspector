# SuJoly Inspector — Спич на 3 минуты

---

## 1. Проблема (20 сек)

«1395 гидротехнических сооружений обследовали в Казахстане в 2025 году. 560 требуют ремонта. В Жамбылской области — сотни каналов, плотин, шлюзов, и каждый имеет паспорт на бумаге. Нет единой карты. Нет актуального статуса. Инспектор едет в поле с папкой.

Мы построили SuJoly Inspector — цифровую операционную платформу, которая это меняет.»

---

## 2. Демо (2 мин 10 сек)

**Карта → Паспорт → Дашборд → HydroFinder → Поле → AI**

### Карта + Паспорт (30 сек)

«Открываем приложение — карта Жамбылской области с 55 сооружениями. Цвет = статус: зелёный исправно, красный — критическое. Кликаем на объект — цифровой паспорт. Четыре вкладки: обзор с техпараметрами, история осмотров, балл риска с разбивкой по компонентам, и документы. Не просто точка на карте — полная карточка объекта.»

### Фильтры + Дашборд (25 сек)

«Фильтрация по району, бассейну, типу, состоянию. Дашборд — аналитика портфеля: круговая диаграмма распределения по состоянию, очередь ремонта по приоритету, статистика осмотров, географическое распределение. Это систематизация и отчётность в одном месте.»

### HydroFinder (25 сек)

«Наша фишка. HydroFinder находит сооружения из внешних источников — OpenStreetMap, спутниковые снимки — которых нет в реестре. Кандидат сравнивается с существующей записью бок о бок: имя, координаты, тип, расстояние. Инженер решает: принять, связать или отклонить. Не ИИ решает — человек решает, система даёт доказательства.»

### Поле офлайн (25 сек)

«Инспектор в поле без интернета. PWA работает офлайн: фото, голосовые заметки на казахском, GPS. Всё сохраняется локально, синхронизация при появлении связи. Голос автоматически транскрибируется в текст. Это не офисная дашборд — это инструмент для реальных выездов.»

### AI-копилот (15 сек)

«Спрашиваешь: "Почему этот объект требует ремонта?" — система отвечает с обоснованием и ссылками на источники. AI объясняет, не решает за инженера.»

---

## 3. Архитектура (20 сек)

«Next.js PWA + MapLibre на фронтенде. FastAPI + PostgreSQL/PostGIS + TiPG для OGC API на бэкенде. MinIO для файлов, Redis для кэша, LangGraph для AI-воркфлоу. Open standards — OGC API, STAC — система интегрируется с государственными ГИС из коробки.»

---

## 4. Финал (10 сек)

«SuJoly Inspector — не демо. Это система, которую можно развернуть в понедельник. Три языка, офлайн, AI с доказательствами. Спасибо.»

## Шпаргалка для Q&A

- **Данные**: 55 mock-объектов по 11 поселениям. Подключение реестра Казводхоза — одна замена query-функции.
- **Мобильное приложение**: PWA — устанавливается на телефон, не нужен App Store.
- **Масштаб**: архитектура на 1400+ объектов. MapLibre + векторные тайлы через TiPG.
- **Ключевая фраза**: «PostGIS — источник истины, LLM не принимает инженерных решений»
